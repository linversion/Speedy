# :core:screenshot-testing 模块设计规范

## 模块定位
`:core:screenshot-testing` 是专门用于**视觉回归测试（截图测试）**的工具层。它封装了底层截图框架（如 Roborazzi），为 UI 组件提供简便的截图比对能力。

## 核心设计逻辑
1. **统一截图配置**：统一定义截图的精度、保存路径和设备模拟参数。
2. **简化 API**：提供易用的扩展函数，让开发者在 `core:ui` 或 `feature` 模块中能一行代码触发截图。

## 准入规则
- **截图框架封装**：底层的截图库配置逻辑。
- **视觉比对工具**：辅助进行图像差异分析的逻辑。

## 测试规范
- **职责**：本模块是 `Screenshot Tests` 的驱动引擎。
- **使用建议**：
    - 开发公共 UI 组件（`:core:ui`）或页面（`:feature:*`）时，应利用此模块提供的工具编写截图测试用例。
    - 配合 `NiaTheme` 验证不同配置下的视觉表现。
