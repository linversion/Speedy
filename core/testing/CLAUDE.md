# :core:testing 模块设计规范

## 模块定位
`:core:testing` 是全项目的**通用测试基石层**。它提供基础的测试类、JUnit Rules 和 Runner，是所有其他模块测试代码的基础依赖。

## 核心设计逻辑
1. **统一运行环境**：提供 `NiaTestRunner`，配置统一的 Robolectric 或 AndroidJUnit4 运行参数。
2. **复用工具**：包含通用的测试规则（如处理协程的 `MainDispatcherRule`）和工具类（如时间/监控模拟）。
3. **跨模块共享**：与 `:core:data-test` 类似，为了让所有模块的测试代码都能引用，放在了 `src/main`。

## 准入规则
- **通用测试基类/规则**：不限于特定业务模型的测试工具。
- **自定义 TestRunner**：配置测试环境的类。
- **通用 Utility**：如用于验证 Compose 状态或协程行为的助手。

## 测试规范
- **职责**：服务于全项目。
- **一致性要求**：所有模块的测试应尽可能继承或使用此模块提供的基础规则，以确保测试环境的一致性。
