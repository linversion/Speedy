# :core:common 模块设计规范

## 模块定位
`:core:common` 是项目的**基础公共工具层**。它包含不具备特定业务属性的工具类、扩展函数和全局常量。

## 核心设计逻辑
1. **基础设施**：提供如协程调度器 (`NiaDispatchers`)、结果处理类 (`Result`) 等全项目通用的基础设施。
2. **高复用、低耦合**：这里的代码应该是高度通用的，不应依赖 `data`, `ui` 或其他业务模块。

## 准入规则
- **协程工具**：调度器注入、Scope 管理。
- **通用 Result 模型**：Loading/Success/Error 的通用封装。
- **纯 Kotlin/Android 扩展**：如 Context 扩展、String 处理等。

## 测试规范
- **测试类型**：**Unit Tests (test)**。
- **测试重点**：
    - **工具函数准确性**：验证纯逻辑转换（如字符串格式化、Result 转换）的正确性。
- **要求**：
    - 带有逻辑分支的公共扩展函数必须有单元测试覆盖。
